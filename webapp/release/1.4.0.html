Release v1.4.0, 1st Feb 2015

Source

At version v1.4.0, 2 bits of code which used to be in core were split out into seperate extensions, "Contact" and "CuratedLists". "CuratedLists" currently must be enabled and added to your config. "Contact" should be enabled and added to your config if you want to use it, but can be left if not.

Key improvements

  • New UI/UX work
  • Sys admin can mark Areas, Venues, Events and Tags as being duplicate. #237 #238 #239 #243
  • Users can add groups to Curated Lists #111
  • Users can Cancel events #253
  • Overhaul method of saving changes #245
  • Google calendar export option to export to existing calendar #286

An incomplete list of minor improvements

  • Sysadmin can delete and undelete Curated Lists #262
  • Google calendar export option to export to existing calendar #286
  • Long descriptions in upcoming events email should be truncated #287
  • Bug fix; in notify emails, change history was not being built properly commit
