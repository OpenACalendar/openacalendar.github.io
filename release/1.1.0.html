Release v1.1.0, not yet

Major changes

  • History includes changes to area.
  • Better history - shows you what fields have changed. (Note there is a new Cron job associated with this.)
  • On create event, check for duplicates and prompt user.
  • Meaningful slugs, eg "1-edinburgh".
  • If supply GB postcode and no lat/lng, lat/lng is set automatically. (Note this is a new extension that must be set up.)
  • Calendar admins can delete and undelete areas.
  • Recur option if event is not in group, that prompts user to add or create group first.

Note the folder structure has changed. Theme assets (CSS, templates, images, javascript) and Extensions are now saved in a different place.

Minor changes

  • Export boxes have more options and better labels.
  • Unsubscribe email headers set.
  • Sysadmin has display config screen that checks for common mistakes.
  • If on Single Site mode, hide visibility from Calendar Admin as that is useless.
  • Login screen should have seperate username or email fields.
  • Calendar improved with fuller title.
  • When recur event in multiple groups new events are now added to all groups.
  • Have physical events feature able to be set by calendar admin.
  • If site only has one country that country is shown in places menu.
